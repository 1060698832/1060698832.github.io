Vue简介

  • 介绍

    动态构建用户界面的渐进式 JavaScript 框架

  • 官网

  • 特点

    1. 遵循 MVVM 模式
    2. 编码简洁, 体积小, 运行效率高, 适合移动/PC 端开发
    3. 它本身只关注UI, 也可以引入其它第三方库开发项目

  • 周边库

    1. vue-cli vue 脚手架

    2. vue-resource

    3. axios

    4. vue-router: 路由

    5. vuex: 状态管理

    6. element-ui: 基于vue 的UI 组件库(PC 端)

Vue模板语法

Vue模板语法有两大类：

  1. 插值语法：
    • 功能：用于解析标签体内容。
    • 写法：语法: ，xxxx 会作为js 表达式解析。
  2. 指令语法：
    • 功能: 解析标签属性、解析标签体内容、绑定事件。
    • 举例：v-bind:href = ‘xxxx’ ，xxxx 会作为js 表达式被解析。
    • 说明：Vue 中有有很多的指令（以v-开头），此处只是用v-bind 举个例子。

Vue中有2种数据绑定的方式：

  1. 单向绑定（v-bind）：数据只能从data流向页面。
  2. 双向绑定（v-model）：数据不仅能从data流向页面，还可以从页面流向data。
  3. 备注：
    • 双向绑定一般都应用在表单类元素上（如：input、select等）。
    • v-model:value 可以简写为v-model，因为v-model默认收集的就是value值。
    • v-model只能应用在表单类元素（输入类元素/有value属性）上。

Vue对象data和el属性的两种写法

  1. el
    • new Vue()的时候配置el属性
    • 先创建Vue实例，随后通过实例对象.$mount(‘#id||.class’)指定el的值
  2. data
    • 对象式
    • 函数式
  3. 由Vue管理的函数，一定不要写箭头函数，一旦写了箭头函数，this就不是vue实例而是windows实例

MVVM模型

  1. M：模型（Model） 对应data中的数据
  2. V：视图（View） 模板
  3. VM：视图模型（ViewModel） Vue实例对象

数据代理

  • 数据代理：通过一个对象代理对另一个对象中属性操作（读/写）

Vue中的数据代理

  1. 通过vm对象来代理data对象中属性操作（读/写）
  2. Vue中数据代理的好处：更加方便操作data中的数据
  3. 基本原理：
    • 通过Object.dafineProperty()把data对象中所有属性添加到vm上
    • 为每一个添加到vm对象上的属性，都指定一个getter/setter
    • 在getter/setter内部去操作（读/写）data中对应的属性

事件处理

  • 事件的基本使用
    1. 使用v-on:xxx 或 @xxx 绑定事件其中xxx是事件名
    2. 事件的回调需要配置在methods对象中，最终会添加到vm对象中
    3. methods中配置的函数，不需要用箭头函数！否则this指向的不是this而是windows
    4. methods中的函数，都是被Vue所管理的函数，this的指向是vm 或 组件的实例对象
    5. @click=”fun” 和 @click=”fun($event)” 效果一致，但后者可以传参
